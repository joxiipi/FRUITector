FRUITector 2.0 — dashboard for an ESP32‑CAM food scanner.

API reference

The dashboard talks to the device over plain HTTP REST calls. Point firmware at these routes:

MethodPathReturns
GET/api/status{ device, firmware, uptime_s, connected }
GET/api/wifi{ ssid, rssi, ip }
GET/api/sensors{ temperature_c, humidity_pct }
GET/streamMJPEG live video (multipart/x-mixed-replace)
GET/captureSingle JPEG snapshot
POST/api/scan{ food_name, category, freshness, confidence, temperature_c, humidity_pct }
POST/api/ledBody { state: "on" | "off" }

Full contract and payload examples are in README.md.
